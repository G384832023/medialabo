「Webコンテンツ作成」の成果

本テーマについて

この授業では情報工学の専門科目の基礎知識を応用して実践的な能力を修得することが目的である。モーション解析・編集ができることやHTML・CSS等を用いてWebページの作成ができることが最大の目標である。

プロジェクト課題について

プロジェクト課題のページ: project.html

私のプロジェクト課題では、世界の天気についての検索結果を表示するページの作成を行いました。世界の国々の種類が多く、それらをユーザー側に入力させるのは使いずらいと感じたのでイベント処理の1種であるプルダウンメニューでユーザー側に選んでもらうように工夫しました。どの国の天気を知ることができるのか1目でわかるので採用しました。 国を選んで検索ボタンを押すことで天気や位置などが表示されます。ページの見やすさにもこだわり、見やすい配色や配置にこだわりました。

数あてゲーム

数あてゲームのページ: kazuate.html

課題3-1(その1)では数当てゲームに1番重要である、答えと予想の判定をするJavaScriptコードを書きました。Javaとは変数の宣言などプログラミングの書き方が違う部分があるので気をつけました。今回はhtmlは書き換えずにコンソールにのみ表示を行いました。

課題4-1(その2)では主に、課題3-1でコンソールに表示させた判定のコードをhtmlに出力しました。それぞれに要素を設定し、代入することで実行時にページに表示することができた。

課題5-1(その3)では、まず初めにhtml文にあったp要素を全て削除し、input要素やbottom要素を使いHTMLにイベントボタンの設置を行いました。最終的に自分の予想する数値をキーボード入力し回答ボタンを押すことで結果の判定を表示するページを作成することができた。

各授業回の学習事項

第1回 WWW と HTML

第1回の授業では、htmlを利用した簡単なWebページの作成方法です。要素について学び、それをうまく組み合わせて演習1-4のようなちょっとしたページを作ることができた。要素にはたくさんの種類があり少しずつ覚えていく必要がある。

演習の成果: 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

第2回の授業では、作成したhtmlのページを装飾するcssを学びました。cssファイルとリンクを繋げることでそれに応じてhtmlに装飾が追加されていく。cssはセレクタやプロパティ・プロパティ名の3つの要素で構成されていて、文字に色をつけたりフォントを変えたりなどたくさんの装飾種類がある。

演習の成果: 練習問題2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

第3回の授業では、Webページ上で複雑な機能を実現するプログラミング言語であるJavaScriptについて学びました。前回学んだページを装飾するcssに加えてページに動きをつけるものがJavaScriptであり、普段私たちが使っているJavaとは少しプログラムの書き方に違いがある。これもcssと同じようにリンクを繋げて使う。

演習の成果: 練習問題3-6: オブジェクトの参照

第4回 DOM と JavaScript

第4回の授業では、DOMというWEBコンテンツを表現するデータ構造について学んだ。DOMとJavaScriptを組み合わせることによって動きのあるWEBページを作ることができる。DOMを使って元々あるHTMLに要素を追加・削除・cssの書き換えなどを行いました。直接htmlを書き換えればいいのでは、と思ったが仕事としてプログラムを書いた時にすごく楽になることがわかりました。

演習の成果: 練習問題4-2~4-5: DOMプログラミング

第5回 イベント処理

第5回の授業では、WEBページ上に動作を付け加えることのできるイベントについて学びました。イベント処理の講義ではボタンを設置してそのページで検索を行うことのできるプログラムを作りました。イベント処理が実行される要素にもたくさんの種類がある。検索するものの要素にも色や時間・電話番号などさまざまな種類がある。

演習の成果: 練習問題5-3: イベントを選択する

授業の感想

私がこの『WEDコンテンツ作成』の授業で苦労した点の1つ目はHTMLについてです。最初にHTML文を見た時、プログラミングで扱っているjavaとは全然違うことに驚きました。PやHなどの文を構成する要素やそれらに名前の区別をつける属性など新しいことの連続で理解するのにとても時間がかかりました。その後文全体を構成するhead要素やbody要素を知り、演習1-4で初めて簡単なページを作成できたときは感動しました。 さらには普段私たちが何気なく押しているリンクの作り方まで学び、リンクを押すことでどのようにしてリンク先に飛んでいくのかを知れた時はとても面白いと感じました。次に難しかったのはCSSです。私は想像力や芸術性がないので、CSSはHTML文を修飾するために用いられるものだと聞いて自分はCSS苦手だなと思いました。CSSの文はHTMLとは別のファイルに書き込み、その後リンクを繋げるものだと聞いて頭が痛くなったのを覚えています。 CSSにも箇条書きのように表示したり・文の色を変えるなどたくさんの装飾する要素があることを知って覚えられるわけがないと絶望していましたが、先生自身もその都度調べながらやっていると聞いて安心しました。特に難しいと感じたのはCSS文を装飾したいHTML文とうまく結びつけないと別の文に色がついてしまったり、とそう言った点に苦労しました。次はJavaScriptです。私は結構javaでプログラミングを書くことが好きなので喜びましたがjavaとは別物と知った時驚きました。 それでも多少は似ているところがありましたが、javaの癖が抜けなく苦労しました。DOMとはHTMLに文を追加したり削除したりできると言うことを学んだ時に最初に思ったことは、そんなことずにHTMLをいじって書き換えたらいいんじゃないかと思いました。しかし今後このようなものを仕事で扱った時によりデータが大きいものだと、直接htmlデータを読み込むよりもDOMを通した方がCPUソースや電力の削減になると聞いて納得しました。 最後にイベント処理で難しいと感じた点は、イベントハンドラと呼ばれるイベント処理をするために1番重要な要素の登録と定義の部分が少し複雑で理解するのに時間がかかりました。イベントの種類にも様々あり1つのイベントを追加するだけでも一苦労だったので、WEDページを作ることは大変なことだと再度実感しました。この6回分の授業を通してたくさんの簡単なWEDページを作ってきて、私は将来こう言ったものを仕事にできるのか不安になったのが正直な感想です。 しかしページがうまく表示された時の感動は忘れないので、この授業での経験はとても大きなものになったと思います。
